## 1. ОПИСАНИЕ ПРОЕКТА

### Название
`fat-loss-landing`

### Описание
Одностраничный мобильный лендинг для продажи электронной книги по похудению от локального блогера. Сайт выполняет одну задачу: конвертирует посетителя в покупателя за счёт эмоционального вовлечения, социального доказательства и триггера срочности.

### Целевая аудитория
- Женщины и мужчины 25–45 лет, Азербайджан, Баку
- Пробовали диеты, но сталкивались со срывами и возвратом веса
- Хотят похудеть без экзотических продуктов и абонементов в спортзал
- Предпочитают контент на русском языке, локальные реалии и продукты

### Бизнес-цели
- **Primary:** Конверсия посетителя в покупку книги
- **Secondary:** Сбор контактов (email/телефон) при интеграции с платёжной системой
- **Трафик:** Мобильные устройства (Instagram, TikTok — основной канал продвижения)

### Критерии успеха (KPI)
- Конверсия страницы ≥ 3% (индустриальный стандарт для инфопродуктов)
- Скорость загрузки: Google Lighthouse ≥ 95 (Mobile)
- Время до CTA-действия ≤ 30 секунд
- Bounce rate ≤ 40%

## 3. ПОЛЬЗОВАТЕЛЬСКИЙ ПУТЬ (User Flow)

```
Посетитель из соцсети
  ↓
Экран 1: Крючок (фото До/После + заголовок)
  ↓
Экран 2: Боль (диеты не работают → эмоциональный триггер)
  ↓
Экран 3: Решение (обложка книги + описание системы)
  ↓
Экран 4: Ценность (рецепты, психология, результат)
  ↓
Экран 5: Призыв (CTA с ограничением по времени)
  ↓
Платёжная форма / цель
```

Каждый экран — это одна мысль на 100vh. Пользователь не перегружен информацией. Скролл раскрывает историю последовательно.

## 5. ТЕХНИЧЕСКИЙ СТЕК

| Компонент | Технология | Обоснование |
|---|---|---|
| **Разметка** | HTML5 (семантическая) | Максимальная производительность, без JS-фреймворков |
| **Стили** | Tailwind CSS (PostCSS) | Минимальный CSS в продакшне, утилитарный подход |
| **Сборщик** | Bun | Скорость, zero-config, встроенный PostCSS |
| **Анимации** | Intersection Observer API (vanilla) | Без внешних библиотек, 0 KB лишнего JS |
| **Изображения** | WebP + lazy loading | Оптимизация производительности |
| **Шрифты** | Inter / San Francisco | Apple-эстетика, системный шрифт для iOS |

### Почему не React/Vue
Проект — одностраничный лендинг без состояния, маршрутизации и интерактивности. JS-фреймворк добавит ~50–100 KB JS, снизив Lighthouse-балл. HTML+Tailwind+vanilla JS даёт 100 баллов производительности.

## 13. ДИЗАЙН-ПРИНЦИПЫ (из .cursor/skills)

> Источники: `skill-ui-design-system/SKILL.md`, `html-slides/SKILL.md`, `animation-patterns.md`, `STYLE_PRESETS.md`

### 13.1 Система дизайн-токенов

Вместо хардкоженных значений — семантические токены. Все цвета, скругления и отступы задаются через Tailwind-токены и CSS-переменные.

| ❌ Плохо | ✅ Хорошо |
|---|---|
| `bg-[#ff00ff]`, `text-[#333]`, `p-[17px]` | `bg-accent`, `text-muted`, `p-4` |
| `rounded-[10px]` | `rounded-xl` (из токенов Tailwind) |
| `w-[123px]` | `w-32` или дизайн-токен |

Исключение — только для layout-х improvisation на мобильных (единичные случаи).

### 13.2 Mobile-first и Touch

- Mobile-first breakpoints: base = `<640px`, затем `sm:`, `md:`, `lg:`
- Минимальный touch-target: `min-h-[44px] min-w-[44px]` для всех интерактивных элементов
- Фокус-состояния: `focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-ring`
- `prefers-reduced-motion` — обязательная поддержка: отключать все анимации для пользователей, которым они мешают

### 13.3 Типографика: Distinctive, не generic

В соответствии с html-slides skill, избегать «AI slop»-шрифтов:

| ❌ Избегать | ✅ Рассмотреть |
|---|---|
| Inter (уже выбран — оправдано Apple-эстетикой) | Playfair Display для акцидентных заголовков |
| Roboto, Arial, system-ui fallback | Space Grotesk для body-текста |
| Однообразные пары | Контрастная пара: гротеск (заголовки) + антиква (акценты) |

**Правило:** Если используем Inter для заголовков (вписывается в Apple-минимализм), то хотя бы для акцентных элементов (цитаты, цифры, CTA) используем контрастный шрифт.

### 13.4 Анимации: Emotion-Driven

Из `animation-patterns.md` — соответствие эффекта и ощущения:

| Ощущение | Анимация | Куда применить |
|---|---|---|
| **Драматичный** | Slow fade-in (1–1.5 с), scale 0.9→1 | Экран 2 (боль), Экран 5 (финал) |
| **Минималистичный / Calm** | Gentle fade, translateY(20px) 0.6s ease-out-expo | Экран 4 (фичи), появление текста |
| **Уверенный / Professional** | Subtle 200–300ms, чёткие transitions | CTA-кнопки, hover-состояния |

**Техника:**
- Использовать Intersection Observer API (уже заложено в PRD)
- CSS `ease-out-expo` для premium-ощущения (Apple-style)
- Staggered entrance с `transition-delay` для списков

**Anti-pattern:** Не анимировать всё подряд — фокус на ключевых моментах: появление экрана, CTA, смена блока.

### 13.5 Anti-Patterns (из skill-ui-design-system)

- ❌ Условные строки классов: `` `${isOpen ? 'block' : 'hidden'}` `` — вместо этого использовать Tailwind-утилиты и JS-управление классами
- ❌ `!important` или глубокие CSS-селекторы для переопределения
- ❌ Хардкод цветов (даже если палитра всего из 3 цветов — зафиксировать их в tailwind.config.js как токены)
- ❌ Игнорирование `prefers-reduced-motion` и фокус-состояний
- ❌ Отрицательные CSS-функции напрямую: `-clamp(28px, 3.5vw, 44px)` — браузер игнорирует. Использовать `calc(-1 * clamp(...))`

### 13.6 Фоны и текстуры (из html-slides)

Для глубины без дополнительных изображений:
- **Gradient mesh:** слоистые радиальные градиенты для Ambient-освещения
- **Noise texture:** полупрозрачный слой шума через inline SVG data-URI
- **Grid pattern:** тонкая сетка (50px) для техничности

Применительно к проекту:
- Экран 1 (Hero): split-screen фото — фон уже есть
- Экран 2 (боль): чёрный — можно добавить тонкий noise overlay для тактильности
- Экран 3 (продукт): белый — мягкий gradient mesh для глубины вокруг обложки
- Экран 4 (фичи): #F5F5F7 — можно добавить тонкую grid pattern
- Экран 5 (CTA): чёрный + фото — минимализм без текстур

### 13.7 Визуальная иерархия и композиция

- **Split-screen** для hero (50/50 фото + текст) — задаёт динамику
- **Крупная изолированная типографика** на чёрном фоне — максимальный контраст и фокус
- **Карточка продукта** на белом — фасилитирует доверие и читаемость
- **Negative space** как инструмент: padding и margin «дышат», текст не прижат к краям
- **CTA-кнопки** — единственные цветные акценты на странице (акцентный цвет, крупные скругления)
