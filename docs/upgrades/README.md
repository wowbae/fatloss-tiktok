# Upgrades

Здесь фиксируются апгрейды и миграции проекта.

Это папка про изменения версий, совместимости и rollout/rollback.
Если изменение не меняет внешние контракты и только улучшает внутренний код, его нужно описывать в `docs/refactoring/`.

Минимум для каждой записи:

- что меняли;
- почему меняли;
- риски;
- шаги отката (если применимо).
