# Refactoring

Здесь фиксируются изменения внутренней структуры кода без добавления новой пользовательской функциональности.

Примеры:

- упрощение архитектуры модулей;
- удаление legacy-кода и техдолга;
- выравнивание типов и импортов;
- улучшение читаемости и поддержки.

Минимум для каждой записи:

- контекст проблемы;
- что изменили;
- почему решение лучше;
- как проверили, что поведение не сломано.
